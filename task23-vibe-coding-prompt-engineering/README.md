\# Task 23 - Vibe Coding and Prompt Engineering using Generative AI



\## 1. What is Vibe Coding?

Vibe Coding is a modern development approach where developers

use natural language prompts to generate code using AI tools

like Claude, ChatGPT, or GitHub Copilot instead of writing

every line manually.



\---



\## 2. Tools Used

\- \*\*Claude AI\*\* (Anthropic) - Primary AI assistant

\- \*\*GitHub Copilot\*\* - Code completion

\- \*\*ChatGPT\*\* - Code generation and debugging



\---



\## 3. Prompt Engineering Techniques Used



\### Technique 1: Clear and Specific Prompts

\*\*Prompt:\*\*

```

Create a Spring Boot REST API for Product Management

with GET, POST, PUT, DELETE endpoints returning JSON responses.

Include H2 in-memory database and JPA repository.

```

\*\*Generated Output:\*\* Complete ProductController.java with all CRUD operations



\---



\### Technique 2: Step-by-Step Prompts

\*\*Prompt:\*\*

```

Step by step create a Eureka Server for microservices:

1\. Show the main class with @EnableEurekaServer

2\. Show application.properties configuration

3\. Show pom.xml dependencies needed

```

\*\*Generated Output:\*\* Complete Eureka Server setup in 3 steps



\---



\### Technique 3: Context-Aware Prompts

\*\*Prompt:\*\*

```

I have an existing identity-service on port 8081 registered

with Eureka. Now create an order-service that calls

identity-service using Feign Client with fallback handling.

```

\*\*Generated Output:\*\* OrderController + UserClient interface + Fallback class



\---



\### Technique 4: Debugging Prompts

\*\*Prompt:\*\*

```

I'm getting this error in Spring Boot:

"Could not find or load main class com.example.demo.Task14EurekaServerApplication"

How do I fix this?

```

\*\*Generated Output:\*\* Step by step fix using Maven Update Project

\### Technique 5: Cloud Configuration Prompts

Create a GitHub Actions CI/CD pipeline YAML for a Spring Boot

Generated using prompt:

"Create a Spring Boot GlobalExceptionHandler with @RestControllerAdvice

that handles ProductNotFoundException with 404 and validation errors with 400"

&#x20;   @ExceptionHandler(ProductNotFoundException.class)

&#x20;   }
